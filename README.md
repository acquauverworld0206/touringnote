
## アプリケーション名

TouringNote

## アプリケーション概要

ツーリングで訪れたい場所を、写真と共に記録・管理するためのWebアプリケーションです。
登録したスポットの中からランダムで行き先を提案する「ランダム抽選機能」も搭載しており、次のツーリング計画を立てる手助けをします。

## 利用方法

1.  **ユーザー登録/ログイン**: トップページから新規登録またはログインを行います。
2.  **スポット登録**: ヘッダーの「新しいスポットを登録」から、スポットの名前、説明、住所、写真などを登録します。
3.  **スポット一覧**: 登録したスポットが一覧で表示されます。
4.  **スポット詳細**: スポット名をクリックすると詳細ページに移動し、登録した情報が表示されます。
5.  **ランダム抽選**: 一覧ページの上部にある「ランダム抽選！」ボタンを押すと、登録済みのスポットの中からランダムで1つが選ばれ、その詳細ページに移動します。
6.  **スポット編集/削除**: 一覧ページまたは詳細ページから、登録したスポットの情報を編集・削除できます。
7.  **グループ機能**: グループを作成し、他のユーザーを招待できます。グループ内でツーリング計画を共有し、候補となるスポットを出し合うことができます。
8.  **投票機能**: グループのメンバーは、候補として挙げられたスポットに対して投票することができます。これにより、民主的に行き先を決定することが可能です。
9. **チャット機能**: 各グループページにはチャット機能が備わっており、メンバー間でリアルタイムにコミュニケーションを取ることができます。

## アプリケーションを作成した背景

私自身がツーリングを趣味としており、これから行きたい場所を簡単に管理したいと思ったのが開発のきっかけです。 
既存のメモや写真を紐付けて管理しにくかったりする課題がありました。
そこで、ツーリングに特化したシンプルな機能で「自分だけのツーリングスポットリスト」を作成・管理できるアプリケーションを開発しました。
また、「どこへ行こうか」と悩む時間も楽しみに変えられるよう、行き先をランダムに提案する機能も加えました。

## 工夫したポイント

*   **ユーザー体験(UX)を考慮した機能設計**:
    単なる情報の記録・管理(CRUD)に留まらず、ツーリングの「行き先に迷う」という課題を解決するために「ランダム抽選機能」を企画・実装しました。これにより、アプリケーションを使う楽しさと、次のツーリングへのきっかけを提供しています。また、グループ機能や投票・チャット機能により、ソロツーリングだけでなくマスツーリングの計画にも対応できる拡張性を意識しました。
*   **モダンな開発環境と効率的なUI構築**:
    バックエンドはRails 7、フロントエンドにはTailwind CSSを採用しました。特にTailwind CSSを用いることで、CSSファイルを肥大化させることなく、迅速かつメンテナンス性の高いUIコンポーネントの構築を実現しました。
*   **位置情報の活用と将来性のある設計**:
    Geocoder gemを導入し、ユーザーが住所を入力するだけで緯度・経度を自動的に取得・保存する機能を実装しました。これにより、ユーザーの手間を省くと同時に、正確な位置情報をデータベースに蓄積できます。この仕組みは、今後Google Maps APIと連携して地図上にスポットを表示するなど、さらなる機能拡張を行うための重要な基盤となっています。
*   **本番環境を意識したデータベース構成**:
    開発環境ではMySQLを、デプロイ先であるRenderで標準的にサポートされているPostgreSQLを本番環境で採用しました。`database.yml`で環境ごとに設定を分離することで、ローカルでの開発のしやすさと、本番環境での安定稼働を両立させる構成を意識しました。

## 改善点

*   **Google API連携によるUXと情報価値の向上**:
    *   **住所入力の改善**: 現在は住所を手入力する必要がありますが、Google Places APIのオートコンプリート機能を導入し、入力の手間を削減すると共に入力ミスを防ぎます。選択された場所の`place_id`を保存することで、データの正確性を向上させます。
    *   **地図表示機能の実装**: スポット詳細ページにGoogle Maps JavaScript APIを連携させ、登録された位置情報をもとに地図を表示します。これにより、ユーザーはスポットの場所を直感的に把握できるようになります。
    *   **スポット情報の拡充**: 保存した`place_id`を活用し、Googleからその場所の公式情報（電話番号、ウェブサイト、評価など）を取得・表示する機能を実装し、アプリケーションの情報価値を高めます。
*   **パフォーマンスの最適化**:
    スポットやグループメンバーが増えた際にN+1問題が発生する可能性があるため、`bullet` gemを導入して問題を検知し、`includes`メソッドを用いて積極的にクエリを最適化していきます。
